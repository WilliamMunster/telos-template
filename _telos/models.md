# Models

> 常用的思维模型，decision-helper 会引用这些模型辅助决策。

## MO1: 闭环验证模型

做任何事之前问：这个东西能跑通一个最小闭环吗？

```
输入 → 处理 → 输出 → 验证 → 反馈
```

如果任何一环断了，先补断点，不要扩展功能。

## MO2: 减法优先

面对选择时，先问"不做会怎样"。如果不做也没事，就不做。

## MO3: 连接 > 重建

优先在已有系统上包一层，而不是从零开始。已有的东西经过了时间验证，新造的没有。
